## 読み上げ記法

3種類の記法があり、それぞれ個別に有効/無効を設定できます。
どの記法でも**コードブロック・インラインコード内は無視**されます。

### 1. 括弧ペア（デフォルト: 『』 で有効）

読み上げ専用の括弧で囲みます。通常の「」会話文には反応しません。

```
『読み上げたいテキスト』
『（ささやき）スタイル指定』
A『キャスト指定（後述）』
```

括弧は好きな開き/閉じ文字列に変更でき、閉じ側に `」.` のような複数文字を
指定すれば「」と衝突しない独自記法も作れます。

日本語の慣習で『』は書名・作品名にも使われるため、素の『』を許すと
作品名で誤爆することがあります。対策は2つ:

- `config brackets` で別の括弧に変える
- `config requireLabel on` で**キャストラベル付きの括弧だけ**読む（誤爆最小・推奨）。
  `A『…』` のような並びは自然文にまず現れないので、素の『』は作品名等に安心して使えます

### 2. `<voice>` タグ（デフォルト: 有効）

複数行をまとめて読ませたいときや、確実に区切りたいときに。
XML風タグが平文に偶然現れることはまずないので、誤爆はほぼゼロです。

```
<voice>読み上げたいテキスト</voice>
<voice style="ささやき">スタイル指定</voice>
```

### 3. 行頭マーカー（デフォルト: 無効）

行頭にマーカーを置くと、その行だけを読み上げます。文中のマーカーには反応しませんが、
装飾として行頭に `♪` 等を書く文章とは衝突するため、デフォルトでは無効です。

```
♪ 読み上げたいテキスト
♪(怒り) スタイル指定もできる
```

### 記法の切り替え

設定は `~/.voicevox-tts.json` に保存され、次の読み上げから即反映されます。

```
voibo-supporter.exe config brackets 《 》     # 括弧を変更（開き 閉じ）
voibo-supporter.exe config brackets 「 」.    # 閉じ括弧+ピリオド方式
voibo-supporter.exe config brackets off      # 括弧記法無効化
voibo-supporter.exe config requireLabel on   # ラベル付き括弧だけ読む（誤爆最小）
voibo-supporter.exe config voiceTag off      # <voice>タグ無効化
voibo-supporter.exe config lineMarker ♪      # 行頭マーカー有効化
voibo-supporter.exe config lineMarker off    # 行頭マーカー無効化
```

### スタイル指定（全記法共通）

キャラクターの別スタイル（声色）に切り替えられます。
使えるスタイル名は接続時に自動検出され、設定に保存されます。

- タグ記法: `style="怒り"` 属性
- 括弧 / 行頭マーカー: 開き括弧・マーカーの直後に `(怒り)` または `（怒り）`

丸括弧の中身が登録済みスタイル名と一致したときだけスタイル指定として扱われます。
一致しない場合（例: `『（笑）そうだね』`）は括弧ごと本文として読まれます。

## キャスト（複数キャラの会話）

大文字1文字のスロット（枠）にキャラクターを登録すると、`A『セリフ』` でそのキャラが
しゃべります。登録数に上限はありませんが、4人程度までが実用的です。

登録するのは**キャラクターだけ**。スタイルは書くたびに小文字添字で選ぶ方式で、
表記と声が必ず一致します（隠れたデフォルト設定はありません）。

```
voibo-supporter.exe cast set A 2     # スロットA = 四国めたん（IDはそのキャラの声ならどれでもよい）
voibo-supporter.exe cast             # 一覧（小文字添字とスタイルの対応表つき）
voibo-supporter.exe cast remove A    # 削除
```

GUI（設定画面のキャストタブ）からも同じことができます。GUI は同じキャストを
1枠だけに置く方式（別の枠で選ぶと移籍）、CLI は同じキャストを複数枠に置く
一人二役もできます。

### 台本での書き方

```
『地の文はいつもの声』
A『こんにちは、めたんよ』        ← 添字なしは a と同じ（1番目のスタイル）
Bb『あまあまスタイルなのだ』     ← 小文字添字でスタイル選択（a=1番目, b=2番目…）
B『（ささやき）スタイル名でも指定できるのだ』
A1『すごく、ゆっくり、読むわよ』  ← 数字添字で読み上げ速度1〜5を指名（2=通常。並びはスタイル→数字）
Bd1『ささやきでゆっくり、なのだ』 ← スタイル添字との合わせ技
A『盛者必衰の｜理《ことわり》』   ← ルビ記法（青空文庫と同じ）。「ことわり」と発音される
```

- 小文字添字とスタイルの対応はキャラごとのスタイル一覧順で、`cast` コマンドか
  GUI のキャスト選択画面で確認できます。範囲外の添字は `a` として読まれます
- スタイル表は**登録時のスナップショット**が正で、勝手に変わりません。
  VOICEVOX の更新でスタイルが増えたときは `cast refresh` で取り直します
- スタイル名の丸括弧指定も併用できます（添字と両方書いた場合はスタイル名が優先）
- 数字添字（1〜5）は[読み上げ速度設定](#読み上げ速度設定速度15)の番号を指します。
  数字つきのセリフは**固定速度**（長文の自動加速や `--speed` より優先）。
  ゆっくり聞かせたい説明文、強調したい一言などに
- ルビ記法 `｜漢字《かな》` で読みを指定できます（`｜` は全角/半角どちらも可）。
  **発音を変えるのは `｜` で振り先を明示したルビだけ**。その場限りの読み指定で、
  全文書共通の読みは[読み替え辞書](#読み替え辞書)の仕事です
- `｜` のない `漢字《かな》`（青空文庫の本文に多い形）は**見た目の注記**扱い:
  `《かな》` を読み飛ばすだけで発音は変えません。青空文庫からコピーした本文を
  二重読みで壊さないための受け皿です。
  ※括弧記法を `《》` に変えている場合はルビ記法と紛れるので注意
- ラベルの直前が英数字の場合は無視されます（英単語の末尾がラベルに化ける誤爆を防止）
- 素の `『』`（ラベルなし）は、先頭のキャスト（通常A）の声で読まれます。
  未登録のラベルも単に無視されて素の `『』` と同じ扱いです
- キャラを増やしても読み上げ時の負荷はほぼ変わりません（増えるのはエンジンのメモリが
  1人あたり数十MB程度と、接続時の予熱1秒×人数だけ）

## 台本プレイヤー（play）

台本ファイル（テキスト）を記法解析して、上から順に再生します。

```
voibo-supporter.exe play 台本.txt
voibo-supporter.exe play 台本.txt --out 出力.wav   # 再生せず1つのWAVに書き出し
voibo-supporter.exe play 台本.txt --speed 1.2      # 速度指定
voibo-supporter.exe play 台本.txt --auto           # 長文なら読み上げ速度の自動調整
```

- 記法が1件も見つからないプレーンテキストは、全行をデフォルト声で読みます
- 次のセリフを裏で合成しながら再生するので、行間の待ちは短めです
- 台本再生は既定では等速（鑑賞用途のため）。長文を早口で流したいときは `--auto`、
  GUI なら設定タブの「台本再生でも読み上げ速度の自動調整を使う」
- 途中でやめるには Ctrl+C、または別の窓から `voibo-supporter.exe stop`
- `--out` で書き出した音声の利用は、**各キャラクターの利用規約（クレジット表記等）に
  従ってください**
- アクセント・間の細かい調整をしたい場合は、本家エディタの「テキスト読み込み」機能が
  向いています。こちらは「今すぐ聞く」ための道具です

## クリップボード読み上げ（paste）

コピーしたテキストを、コマンド実行時に**1回だけ**読み取って読み上げます。
台本プレイヤーと同じ扱いで、記法があれば配役つき、なければ全行デフォルト声。

```
voibo-supporter.exe paste                  # クリップボードの内容を読み上げ
voibo-supporter.exe paste --out 出力.wav   # 再生せずWAVに書き出し
voibo-supporter.exe paste --speed 1.2      # 速度指定
```

- クリップボードの**常時監視はしない**設計です（読むのは実行した瞬間の1回だけ）
- 台本プレイヤーと同様、文字数の自動加速はかかりません（等速）

## 他ソフトからの流し込み（serve）

ローカル HTTP サーバーとして常駐し、送られてきたテキストを読み上げます。
HTTP でテキストを POST できるものなら、どんなソフト・スクリプトからでも使えます。

```
voibo-supporter.exe serve                 # 待ち受け開始（デフォルト http://127.0.0.1:51021）
voibo-supporter.exe serve --port 58000    # ポート指定（恒久設定は config servePort）
voibo-supporter.exe serve stop            # 別の窓から終了（前面の Ctrl+C でも可）
```

| メソッド / パス | 説明 |
|------|------|
| `POST /talk` | リクエスト本文（UTF-8 テキスト）を読み上げ。台本記法もそのまま通る |
| `GET /talk?text=…` | 短文用。ブラウザのアドレス欄や curl 一発で試せる |
| `GET /status` | 動作確認（エンジンの状態を JSON で返す） |
| `POST /stop` | 再生停止（CLI の `stop` と同じ） |

```
curl -X POST --data-binary "A『こんにちは』" http://127.0.0.1:51021/talk
curl "http://127.0.0.1:51021/talk?text=%E3%81%93%E3%82%93%E3%81%AB%E3%81%A1%E3%81%AF"
```

- 受理した時点で応答が返り（202）、合成・再生は裏で進みます。
  同時に流し込まれても音は重なりません（順番待ちで直列再生）
- 話速はクエリ `?speed=1.2` で固定できます。未指定なら文字数の自動加速が効きます
- バインドは **127.0.0.1 固定**（外部には公開されません。LAN 公開機能はあえて持ちません）
- 棒読みちゃんとは連携も競合もしない独自 API です（ポート 50001/50080 には触れません）

## 読み上げ速度設定（速度1〜5）

話速は「速度1〜5」の5つの設定で管理します。**速度2が「通常」**（普段の通知・台本再生の
速さ）で、速度1はゆっくり枠、速度3〜5は速い側。初期値:

| 速度 | 話速 | 使われかた |
|---:|------|-----------|
| 1 | ×0.8 | 数字記法（`A1『…』`）専用のゆっくり枠 |
| 2 | ×1.3 | **通常**（`config speed` と同じもの） |
| 3 | ×1.4 | 80字以上の通知で自動／数字記法 |
| 4 | ×1.55 | 200字以上の通知で自動／数字記法 |
| 5 | ×1.7 | 400字以上の通知で自動／数字記法 |

長文が自動で早口になるのは通知・ながら聞き向けの機能で、台本再生（play/paste）は既定では
等速です。台本にも効かせたいときは `--auto`（設定タブの同名項目と同じ）。
話速を明示指定したとき（`--speed` や serve の `?speed=`）は自動では速くなりませんが、
数字添字のセリフだけはどんな場面でも指定の速度で読まれます。

```
voibo-supporter.exe config speedSteps 0.8 1.3 1.4 1.55 1.7  # 速度1〜5の話速
voibo-supporter.exe config stepChars 80 200 400             # 速度3〜5に切り替わる文字数
voibo-supporter.exe config stepChars off                    # 文字数による自動早口をやめる
```

設定GUI（設定タブ）からも、各速度の話速・しきい値とも変更できます。

1セグメント500文字を超える場合は切り詰められます（上限は設定タブで変更可）。

## 読み替え辞書

VOICEVOX が読み間違える語を、合成直前にクライアント側で文字置換して直す仕組みです。
**VOICEVOX 本体のユーザー辞書とは別物**で、エンジンには何も登録しません
（このツールを通る音声にだけ効きます。アクセント調整はできない、単純な表記置換です）。

- 汎用の読み替え（例: 身体→からだ）はコードに内蔵
- 個人的な読み替えは `~/.voicevox-tts-readings.json` に保存され、内蔵分より優先

```
voibo-supporter.exe reading                  # 一覧表示
voibo-supporter.exe reading add 表記 よみ     # 追加（よみはひらがな推奨）
voibo-supporter.exe reading remove 表記      # 削除
```

## トレイ常駐と設定画面

### トレイ常駐

`voibo-supporter.exe` のダブルクリックでトレイ常駐＋設定画面が開きます。
トレイだけ欲しい場合は `voibo-supporter.exe --tray`。

ダブルクリックで設定画面を開きます。右クリックでメニュー:
読み上げ ON/OFF ／ 再生を停止 ／ serve 待ち受け ON/OFF ／
PC起動時に自動で立ち上げる ／ 設定... ／ 終了

- 「PC起動時に自動で立ち上げる」は、スタートアップフォルダ（`shell:startup`）に
  ショートカットを置く／消すだけの素直な方式。手でショートカットを消しても矛盾しません
- 二重起動しても2つ目は案内を出して終了します
- Explorer が再起動してもアイコンは自動で載せ直されます

### 設定画面

トレイメニューの「設定...」または `voibo-supporter.exe --gui` で開きます。

- **キャスト**タブ: A〜H の8枠。カードをクリックすると選択画面（全キャストのサムネイル
  格子と立ち絵プレビュー・台本での書き方つき）、カードのドラッグ＆ドロップで枠の入れ替え。
  操作は即保存（保存ボタンなし）。下部に選択中キャスト全員の利用規約まとめを表示
- **台本**タブ: 台本の編集・再生・WAV書き出し。テキストファイルの読み込みにも対応
- **設定**タブ: 話速・長文の自動加速・1セグメントの上限文字数・serve のポート・記法・
  エンジン自動起動
- キャラクターの画像・名前・規約文は、**ユーザー自身がインストールした VOICEVOX エンジンの
  公式 API（`/speakers`, `/speaker_info`）から実行時に取得して表示するだけ**で、
  このツールには一切同梱しません
- VOICEVOX が見つからないときは公式サイトへの案内を表示します。
  動作検証済みは **VOICEVOX 0.25.2 以上**で、それより古い場合は更新の案内を出します

## Claude Code 連携の詳細

`install-hook` は Claude Code のグローバル設定（`~/.claude/settings.json`）に
Stop フックを追記します（既存設定は保持し、書き換え前にバックアップを作成。二重登録は
しません）。手動で設定する場合は以下を追加します（パスは環境に合わせる）:

```json
"hooks": {
  "Stop": [
    {
      "matcher": "",
      "hooks": [
        {
          "type": "command",
          "command": "\"C:/…/voibo-supporter.exe\" --hook"
        }
      ]
    }
  ]
}
```

- フックは記法の検出だけして即終了します（0.1秒台）。時間のかかる合成・再生は
  切り離された独立プロセスが担うので、Claude Code の応答処理を待たせません
- エンジン未起動のときは自動起動を試み、VOICEVOX が見つからなければ何も起きません（無害）。
  自動起動は `config autoStart off` で無効化できます
- AI への記法の伝え方: 設定画面のキャストタブに「連携用の共有文言」があり、キャストを
  保存するたびに詳細ガイド（`VOICEVOX_SUPPORTER_AI_GUIDE.md`）が exe の隣に自動生成されます

### Codex デスクトップアプリ

通常の利用に Codex の Stop フック設定は必要ありません。キャストタブの「連携用の共有文言」を
Codex に渡すと、exe の絶対パス・現在の配役・記法が書かれた自動生成ガイドを参照できます。
Codex は返答本文を `A『…』` のような記法で先に完成させ、その同じ本文をガイド記載の
`voibo-supporter.exe --bg` へ標準入力で渡します。読み上げ専用の別原稿は作りません。

この経路は配布exeだけで動作し、Pythonは不要です。Codexがローカルコマンドの実行許可を
求めた場合は、ガイドに表示されたぼいぼサポーターのexeであることを確認して許可してください。

ソースツリーの `codex_tts_hook.py` は開発者向けの任意アダプタです。Codex公式の Stop イベント
（JSON標準入力）と外部通知 `notify`（JSON引数）の最終assistant本文を `speak.py --hook` へ渡します。
通常の配布exe利用では不要で、Stopフックの実行と信頼承認を確認できた環境だけで使用します。

### Claude Code 以外の AI・ツールで使う

本体はただの CLI + VOICEVOX ENGINE クライアントなので、テキストさえ渡せれば
どこからでも使えます:

```
echo "読み上げたいテキスト" | voibo-supporter.exe
voibo-supporter.exe "読み上げたいテキスト"
voibo-supporter.exe --bg "読み上げたいテキスト"   # すぐ返ってほしい呼び出し（AI・スクリプト等）
```

常駐させて HTTP で受けたい場合は [serve](#他ソフトからの流し込みserve)。

自動読み上げの接着部分（`--hook`）だけが Claude Code の Stop フック仕様
（stdin の JSON 内 `last_assistant_message` フィールド）に依存しています。
応答テキストを外部コマンドに渡せるイベント機構を持つツールなら、薄いアダプタで対応できます。

## CLI リファレンス

コマンドは `voibo-supporter.exe <コマンド>`（ソース版は `python speak.py <コマンド>`）。

| コマンド | 説明 |
|---------|------|
| `start` | エンジン接続確認 + 読み上げ有効化 + スタイル自動検出 + キャスト予熱 |
| `on` / `off` | 読み上げの切り替え |
| `stop` | 再生中の音声を停止（合成中なら中断） |
| `status` | 状態確認（有効な記法・キャスト・辞書件数含む） |
| `doctor` | 環境診断（Python・エンジン・設定・フックの確認。何も変更しない） |
| `install-hook` | Claude Code の Stop フックを自動設定（冪等・バックアップ付き） |
| `config` | 設定表示 |
| `config speaker <ID>` | デフォルト声の変更 |
| `config speed <0.5-2.0>` | 通常の話速（=速度2）の変更 |
| `config speedSteps <速度1〜速度5>` | 速度1〜5の話速（例: `0.8 1.3 1.4 1.55 1.7`） |
| `config stepChars <文字数>... \| off` | 速度3〜5に切り替わる文字数（例: `80 200 400`） |
| `config servePort <1024-65535>` | serve のポート変更（デフォルト 51021） |
| `config brackets <開き> <閉じ> \| off` | 括弧記法の設定 |
| `config requireLabel on \| off` | ラベル必須モード（ラベル付き括弧だけ読む） |
| `config voiceTag on \| off` | タグ記法の切り替え |
| `config lineMarker <文字> \| off` | 行頭マーカー変更/無効化 |
| `config autoStart on \| off` | エンジン自動起動の切り替え |
| `cast set <ラベル> <ID>` | キャスト登録（ラベルは大文字1文字） |
| `cast` / `cast remove <ラベル>` | キャスト一覧 / 削除 |
| `cast refresh` | スタイル表をエンジンから取り直す |
| `reading add <表記> <よみ>` | 読み替え辞書に追加 |
| `play <台本.txt> [--out 出力.wav]` | 台本を順次再生 / WAV書き出し |
| `paste [--out 出力.wav]` | クリップボードを読み上げ / WAV書き出し |
| `serve [--port N]` / `serve stop` | ローカルHTTP待ち受けの開始 / 終了 |
| `-l` | キャラ一覧 |
| `"テキスト"` | 手動読み上げ（再生が終わるまで返らない） |
| `--bg "テキスト"` | 切り離し読み上げ（すぐ返る・呼び出し元が終了しても再生継続） |
| `--tray` / `--gui` | トレイ常駐だけ / 設定画面だけを開く |

## 設定ファイル

すべてホームディレクトリ直下（インストールフォルダの外）:

- `~/.voicevox-tts.json` — スピーカーID・速度・記法・スタイル・キャスト等の設定
- `~/.voicevox-tts-readings.json` — 個人用の読み替え辞書
- `~/.voicevox-tts-enabled` — フラグファイル（存在すれば読み上げ有効）

## 音声向きの書き方のコツ

- 感嘆詞の表記を調整する（「んんっ」→「んー」等、VOICEVOX が読みやすい形に）
- 絵文字は入れない（読み上げに影響しないが無駄）
- テキスト出力の丸コピーではなく、音声向けに書き下ろすと聞きやすい
